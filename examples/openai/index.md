# Organization Engineering Style Wiki

Generated by [gitstyle](https://github.com/edgarpavlovsky/gitstyle) from the public repositories of the [openai](https://github.com/openai) GitHub organization.

## Organization Profile

OpenAI's open-source footprint splits along a fault line that runs through nearly every style decision in this wiki. On one side: **Stainless-generated production SDKs** (openai-python, openai-node) that inherit Stripe's resource architecture, strict typing, and exhaustive error hierarchies — conventions chosen by Stainless, not OpenAI engineers. On the other: **hand-crafted research code** (Whisper, CLIP, gym) written in idiomatic PyTorch, optimized for paper reproducibility over production hardening. A third category, **performance-critical tooling** (tiktoken, Triton), bridges both worlds — tiktoken pairs a Rust core with a pure-Python educational reference implementation, while Triton is an MLIR/LLVM compiler project that shares almost nothing stylistically with the rest of the org.

The most interesting patterns are the ones that cross the divide: pip-installable packaging, flat dependency trees, and a preference for raw primitives over frameworks. Articles note these convergences and divergences where they arise.

The most distinctive individual artifact may be tiktoken's `_educational.py` — a pure-Python BPE reimplementation shipped alongside the production Rust core, functioning as living documentation of the algorithm. It captures something about the org's dual identity: serious about performance, serious about making things understandable.

**Primary languages:** Python, TypeScript, C++, Rust
**Licenses:** MIT, Apache-2.0
**Key repos:** openai-python, openai-node, tiktoken, whisper, CLIP, gym, openai-cookbook, triton
