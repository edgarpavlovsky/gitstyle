# Organization Engineering Style Wiki

Generated by [gitstyle](https://github.com/edgarpavlovsky/gitstyle) from the public repositories of the [openai](https://github.com/openai) GitHub organization.

## Organization Profile

OpenAI maintains two distinct categories of open-source code with markedly different engineering standards. The **production SDKs** (openai-python, openai-node) are auto-generated via Stainless and exhibit Stripe-influenced design: resource-based architecture, strict typing, exhaustive error hierarchies, and comprehensive test suites. The **research code** (whisper, CLIP, gym) is PyTorch-based, experiment-oriented, and optimized for reproducibility over production hardening. A third category, **performance-critical tooling** (tiktoken, triton), bridges both worlds with Rust/C++ cores and Python bindings.

This dual nature means many style conventions have explicit exceptions depending on which category a repository belongs to. Articles note these contrasts where they arise.

**Primary languages:** Python, TypeScript, C++, Rust
**Licenses:** MIT, Apache-2.0
**Key repos:** openai-python, openai-node, tiktoken, whisper, CLIP, gym, openai-cookbook, triton
