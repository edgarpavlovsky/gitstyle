# Org Engineering Style Wiki

Generated by [gitstyle](https://github.com/edgarpavlovsky/gitstyle) from the public repositories of the [anthropic](https://github.com/anthropics) organization.

## Organization Profile

Anthropic is an AI safety company building frontier language models (Claude) and the developer tools that expose them. The public engineering surface is dominated by two official SDKs (Python and TypeScript), a cookbook of integration examples, educational course materials, and a CLI agent. The organization prioritizes developer experience, strong typing, and streaming-first API design across all repositories. All public code is MIT-licensed.

## Style Dimensions

- [[code-structure]] — File and module organization: resource-based client architecture, mirrored SDK layouts, and explicit export surfaces
- [[naming-conventions]] — Naming patterns: PascalCase models, snake_case methods, prefixed error hierarchies, and `__all__` exports
- [[patterns]] — Error handling, async-first design, streaming SSE architecture, and Pydantic-driven request/response modeling
- [[type-discipline]] — Typing strictness: comprehensive type hints in Python, strict mode in TypeScript, and generic resource typing
- [[testing]] — Test philosophy: pytest with parametrize, mock HTTP layers, snapshot testing, and streaming replay fixtures
- [[comments-and-docs]] — Documentation patterns: docstrings on public API, cookbook-driven tutorials, and inline explanations of protocol-level details
- [[dependencies]] — Preferred tools and libraries: httpx, Pydantic, node-fetch, and the minimal-dependency principle
- [[commit-hygiene]] — Commit and PR patterns: conventional-ish prefixes, changelog-driven releases, and CI-gated merges

## Language-Specific

- [[languages/python]] — Python SDK idioms: `@override`, `__all__`, Pydantic models, httpx transport
- [[languages/typescript]] — TypeScript SDK idioms: strict mode, Zod-like runtime validation, branded types, and API mirroring
