# Org Engineering Style Wiki

Generated by [gitstyle](https://github.com/edgarpavlovsky/gitstyle) from the public repositories of the [anthropic](https://github.com/anthropics) organization.

## Organization Profile

Anthropic builds frontier language models (Claude) and the developer tools that expose them. The public engineering surface centers on two official SDKs (Python and TypeScript) generated by [Stainless](https://www.stainlessapi.com/), a cookbook of integration examples, educational course materials, and a CLI agent. Because the SDKs are Stainless-generated, many structural patterns — resource-based architecture, sync/async mirroring, error hierarchies — are shared conventions with other Stainless clients (notably OpenAI's SDKs). Anthropic's distinctive engineering choices show up in the cookbook's educational design, the streaming-first API surface, and the strict typing discipline applied on top of the generated scaffold. All public code is MIT-licensed.

## Style Dimensions

- [[code-structure]] — File and module organization: Stainless resource-based client architecture, mirrored SDK layouts, and explicit export surfaces
- [[naming-conventions]] — Naming patterns: PascalCase models, snake_case methods, prefixed error hierarchies, and `__all__` exports
- [[patterns]] — Error handling, streaming SSE architecture, retry semantics, and tool use modeling
- [[type-discipline]] — Typing strictness: exhaustive type hints in Python, strict mode in TypeScript, discriminated unions, and the BaseModel/TypedDict split
- [[testing]] — Test philosophy: pytest with parametrize, HTTP transport mocking, snapshot testing, and streaming replay fixtures
- [[comments-and-docs]] — Documentation patterns: generated docstrings, cookbook-driven tutorials, and protocol-level inline comments
- [[dependencies]] — Preferred tools and libraries: httpx, Pydantic, minimal dependency surface, and modern Python project management
- [[commit-hygiene]] — Commit and PR patterns: conventional-ish prefixes, changelog-driven releases, and CI-gated merges

## Language-Specific

- [[languages/typescript]] — TypeScript SDK idioms: strict mode, async iterables, discriminated unions, branded types, and dual ESM/CJS publishing
